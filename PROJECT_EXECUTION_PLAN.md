## 🎯 项目目标

### 总体目标
完成 AcademicHelper macOS 应用的 MVP 版本，具备完整的用户界面、稳定的测试覆盖和发布准备。

4. **用户体验**: 通过内部测试，无明显 bug


### 应急预案

**情景: Phase 1 延期**
- 触发条件: Day 7 时完成度 < 80%
- 应对措施:
  1. 削减非核心功能 (如 AIAssistantView 简化版)
  2. 增加每日工时
  3. 调整后续阶段时间

**情景: 测试发现严重 Bug**
- 触发条件: 阻塞性 Bug > 3 个
- 应对措施:
  1. 启动 Bug 修复冲刺
  2. 暂停新功能开发
