# AcademicHelper 项目规划方案

## 1. 项目背景与目标

### 1.1 项目背景
AcademicHelper 是一款面向 macOS 平台的学术辅助应用，旨在解决学术工作者在文献阅读、词汇积累和学术写作中遇到的效率问题。当前市场上的工具多为单一功能，缺乏集成化的解决方案。

### 1.2 项目目标
- **短期目标（3个月）**：完成 MVP 版本，包含单词学习和文献管理核心功能
- **中期目标（6个月）**：完善写作辅助模块，实现 iCloud 同步
- **长期目标（12个月）**：建立学术社区，支持用户分享表达库和术语库

### 1.3 目标与愿景
**愿景**：成为学术界首选的智能化辅助工具，让知识获取和学术写作更加高效。

**核心价值观**：
- 效率至上：减少重复劳动，聚焦核心研究
- 知识连接：建立单词-文献-表达的知识网络
- 持续学习：科学的记忆算法助力长期积累

## 2. 核心任务与实施步骤

### 2.1 功能需求规格

#### 2.1.1 单词学习模块
| 功能 | 优先级 | 描述 |
|------|--------|------|
| 屏幕取词 | P0 | 全局监听选中文本，弹出词典释义 |
| 词典查询 | P0 | 集成第三方词典API，支持离线缓存 |
| 生词本管理 | P0 | 增删改查，分类管理 |
| SM-2复习 | P0 | 科学的间隔重复算法 |
| 复习提醒 | P1 | 本地通知提醒待复习单词 |
| 学习统计 | P1 | 学习时长、掌握度分析 |

#### 2.1.2 文献管理模块
| 功能 | 优先级 | 描述 |
|------|--------|------|
| PDF导入 | P0 | 支持拖拽和文件选择器导入 |
| PDF阅读 | P0 | 内置PDF阅读器，支持缩放和导航 |
| 元数据提取 | P0 | 自动提取标题、作者、页数等信息 |
| 词-文献关联 | P0 | 建立单词与文献的双向关联 |
| 文本提取 | P1 | 提取PDF文本内容 |
| 文献标注 | P2 | 高亮、笔记功能 |

#### 2.1.3 写作辅助模块
| 功能 | 优先级 | 描述 |
|------|--------|------|
| 表达识别 | P1 | 识别学术表达并分类 |
| 表达库 | P1 | 收藏和管理学术表达 |
| 智能推荐 | P2 | 根据上下文推荐替代表达 |
| 术语库 | P2 | 计算机科学专业术语 |

## 5. 风险评估与应对策略

### 5.1 技术风险

| 风险 | 概率 | 影响 | 应对策略 |
