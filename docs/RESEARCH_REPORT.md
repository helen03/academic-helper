# AcademicHelper 调研报告

## 1. 项目概述

### 1.1 项目背景
AcademicHelper 是一款专为 macOS 平台设计的学术辅助应用，旨在帮助研究人员、学生和学术工作者更高效地进行文献阅读、词汇学习和学术写作。

### 1.2 目标用户
- 高校研究人员和博士生
- 硕士研究生和本科生
- 需要阅读大量英文文献的学术工作者
- 希望提升学术英语写作能力的用户

## 2. 竞品分析

### 2.1 现有解决方案

#### 2.1.1 欧路词典 (Eudic)
**功能特点：**
- 屏幕取词和划词翻译
- 多词典支持
- 生词本管理
- 跨平台同步

**局限性：**
- 缺乏文献管理功能
- 无法建立单词与文献的关联
- 复习算法较为简单
- 没有学术表达识别功能

#### 2.1.2 Zotero
**功能特点：**
- 强大的文献管理
- PDF 阅读和标注
- 引用管理
- 浏览器插件支持

**局限性：**
- 缺乏单词学习功能
- 无法识别和收集学术表达
- 界面较为传统

#### 2.1.3 Papers (ReadCube)
**功能特点：**
- 现代化界面设计
- 智能推荐
- 云同步
- 协作功能

**局限性：**
- 订阅制收费
- 缺乏单词学习模块
- 学术表达功能缺失

### 2.2 市场空白
通过竞品分析，我们发现市场上缺乏一款**集成化**的学术辅助工具，能够同时满足：
1. 屏幕取词和词汇学习
2. 文献管理和阅读
3. 学术表达收集和推荐
4. 专业领域术语支持

### 3.2 技术栈选择

| 层级 | 技术 | 说明 |
|------|------|------|
| UI | SwiftUI + AppKit | 现代化声明式UI，兼容macOS原生组件 |
| 架构 | MVVM + 依赖注入 | 清晰的关注点分离，便于测试 |
| 数据持久化 | Core Data + CloudKit | 本地存储 + 云端同步 |
| PDF处理 | PDFKit | 系统原生PDF渲染和解析 |
| 屏幕取词 | Accessibility API | 系统级文本捕获 |
| 词典服务 | Free Dictionary API | 免费开源词典API |
| 复习算法 | SM-2算法 | 科学的间隔重复算法 |

### 3.3 核心模块设计

#### 3.3.1 单词学习模块

**屏幕取词流程：**
```
用户选中文本 → Accessibility API 捕获 → 词典查询 → 弹窗显示 → 加入生词本
```

**SM-2 复习算法：**
```
间隔天数 = 上一次间隔 × 难度系数
难度系数 = 原系数 + (0.1 - (5-质量分)×(0.08+(5-质量分)×0.02))
```

#### 3.3.2 文献管理模块

**PDF处理流程：**
```
导入PDF → 元数据提取 → 文本解析 → 建立索引 → 存储到数据库
```

**词-文献关联：**
```
单词 ←──多对多──→ 文献
  │                │
  └── 在文献中遇到的单词
```

#### 3.3.3 写作辅助模块

**学术表达识别：**
```
用户输入文本 → 正则匹配 → 分类识别 → 推荐替代表达
```

## 5. UI设计建议

### 5.1 设计原则
- **简洁专业**：学术工具应注重内容，避免过度设计
- **高效操作**：减少点击次数，支持快捷键
- **视觉层次**：清晰的信息层级，便于快速扫描
- **一致性**：遵循macOS人机界面指南

### 5.2 配色方案
- 主色调：系统蓝色 (#007AFF)
- 背景色：系统背景色（支持深色模式）
- 强调色：橙色 (#FF9500) 用于复习提醒
- 文字色：系统标签色

## 6. 数据存储与同步

### 6.1 本地存储
- **Core Data**：结构化数据（单词、文献、表达）
- **文件系统**：PDF文件、用户设置
- **Keychain**：敏感信息（如API密钥）

### 6.2 云端同步
- **CloudKit**：苹果原生云服务，与iCloud集成
- **同步策略**：
  - 实时同步：单词添加、文献导入
  - 定时同步：复习记录、统计信息
  - 冲突解决：以最新修改为准

### 6.3 备份策略
- 自动每日备份到iCloud
- 支持手动导出/导入数据

## 7. 潜在技术难点及解决方案

### 7.1 屏幕取词准确性
**难点：** 不同应用的文本渲染方式不同，Accessibility API可能无法捕获

**解决方案：**
- 使用多种检测方式（AX API + 剪贴板监听）
- 提供手动输入备选方案
- 建立常见应用的白名单

### 7.2 PDF文本提取
**难点：** 扫描版PDF无法直接提取文本

**解决方案：**
- 集成Vision框架进行OCR识别
- 提示用户PDF类型（文本/扫描）
- 支持手动标注关联

### 7.3 数据同步冲突
**难点：** 多设备同时修改同一数据

**解决方案：**
- 使用时间戳和版本号
- 实现冲突检测和手动合并界面
- 优先保留用户明确操作的数据

### 7.4 性能优化
**难点：** 大量PDF和单词数据时的性能

**解决方案：**
- 分页加载和虚拟列表
- 后台线程处理PDF解析
- 数据库索引优化

## 9. 总结

AcademicHelper 通过集成屏幕取词、文献管理和写作辅助三大核心功能，填补了当前学术工具市场的空白。采用现代化的技术栈和科学的复习算法，能够显著提升用户的学术研究效率。

**核心竞争优势：**
1. 一站式学术工作流
2. 智能的词-文献关联
3. 科学的记忆算法
4. 原生macOS体验
5. 隐私优先（本地处理）
